# Introducing the MCP Catalog in Red Hat OpenShift AI: 10 MCP servers ready to deploy

AI agents are only as useful as the tools they can reach. The Model Context Protocol (MCP) has rapidly become the standard for connecting AI agents to enterprise systems — but finding trusted, production-ready MCP servers and deploying them on your infrastructure has meant stitching together unvetted servers from scattered sources. That changes today.

With Red Hat OpenShift AI 3.4, we are introducing the **MCP Catalog** in Developer Preview — a curated catalog of MCP servers that you can discover, deploy and manage directly on Red Hat OpenShift. It ships pre-loaded with 10 MCP servers spanning Red Hat products, technology partners and open source community projects. **[Try Red Hat OpenShift AI](https://www.redhat.com/en/technologies/cloud-computing/openshift/openshift-ai/trial)** to explore the MCP Catalog yourself.

This is not a static listing. Every MCP server in the catalog deploys live on your cluster through the MCP Lifecycle Operator, also shipping in Developer Preview. And this is just the beginning of the enterprise MCP ecosystem we are building.

## The MCP Catalog: from discovery to deployment

Most MCP catalogs available today are discovery surfaces — they help you find servers but leave deployment, security and lifecycle management up to you. The MCP Catalog in Red Hat OpenShift AI takes a different approach.

When you browse the catalog in the AI Hub, you are looking at MCP servers that have been validated for enterprise deployment. Each server meets a defined set of technical requirements: streamable HTTP transport, on-cluster deployment and UBI-based container images scanned for vulnerabilities. When you select a server, the MCP Lifecycle Operator handles deployment — creating the Kubernetes resources, exposing the service and making the server's tools available to your AI workloads in Gen AI Studio.

The result is a governed path from discovery to deployment. No manual container wrangling. No guessing whether an MCP server will work in your environment. Select, deploy, connect.

## 10 MCP servers ready to use

The catalog ships with three tiers of MCP servers, each addressing real enterprise workflows.

**Red Hat MCP servers** connect your AI agents to the platforms you already run:

- **Red Hat OpenShift** — Agents can query cluster state, manage workloads and troubleshoot deployments directly through natural language.
- **Red Hat Ansible Automation Platform** — Connect agents to your automation workflows. An agent can trigger Ansible playbooks, check job status and orchestrate configuration changes across your infrastructure.
- **Red Hat Insights (Lightspeed)** — Surface platform intelligence and recommendations through your AI agents, bringing operational insights into agentic workflows.

**Technology partner MCP servers** extend your agents into the broader enterprise stack:

- **Confluent Kafka** — Give agents the ability to interact with event streams, manage topics and monitor your real-time data pipelines.
- **EnterpriseDB pg-airman-mcp** — Connect agents to PostgreSQL databases for queries, schema management and database operations.
- **HashiCorp Terraform** — Let agents provision and manage infrastructure as code, bridging the gap between AI-driven decisions and infrastructure execution.
- **Microsoft Azure** — Enable agents to manage Azure resources and cloud operations, complementing your hybrid cloud strategy on OpenShift.
- **Dynatrace** — Bring full-stack observability into your agentic workflows. Agents can query performance data, investigate incidents and correlate metrics across your environment.

**Community MCP servers** round out the data tier:

- **MongoDB** — Document database operations for agents working with unstructured and semi-structured data.
- **MariaDB** — Relational database connectivity for agents that need to query and manage structured data.

## Building the enterprise MCP ecosystem

The 10 MCP servers shipping in RHOAI 3.4 are the foundation — not the ceiling. We are building toward a full enterprise MCP ecosystem with the capabilities that production deployments demand.

**Quickstarts are coming.** We are working with our partners to deliver guided quickstart experiences that walk you through real-world use cases — from connecting Dynatrace observability into your agentic incident response workflows to managing Azure resources alongside your OpenShift workloads. Expect the first quickstarts shortly after Red Hat Summit.

**The ecosystem will grow.** We are actively onboarding new partners and expanding the catalog. The process we have built — from partner curation and consent through technical validation and catalog integration — is designed to scale. MCP servers are the first AI asset type going through this pipeline and we will use the same model for agents, skills and other AI assets as they mature.

**Enterprise governance is on the roadmap.** The MCP Catalog is the discovery and deployment layer. In upcoming releases, we are adding the governance backbone: supply chain management, trust tiering and certification, access control, lifecycle state management and full auditability. Our goal is to give platform engineers and AgentOps teams the same level of control over MCP servers that they expect for any production workload.

This is what an enterprise MCP ecosystem looks like — not just a place to find servers, but a platform to govern, deploy and operate them with confidence.
