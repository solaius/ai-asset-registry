# Blog Abstract: The MCP Catalog arrives in Red Hat OpenShift AI

## Thesis

Red Hat OpenShift AI 3.4 introduces the MCP Catalog in Developer Preview — pre-loaded with 10 MCP servers from Red Hat, technology partners and the open source community — marking the beginning of an enterprise MCP ecosystem built on trust, governance and lifecycle management.

## Target Audience

IT decision makers, enterprise architects and platform engineers evaluating how to bring AI agent tooling into production with enterprise-grade governance and partner ecosystem support.

## Blog Type

Red Hat Blog (redhat.com/en/blog) — thought leadership / announcement

## Key Points (3 maximum)

1. **The MCP Catalog is here**: RHOAI 3.4 ships a curated MCP Catalog backed by the MCP Lifecycle Operator, giving teams a governed path to discover and deploy MCP servers on OpenShift — not just a list, but a live-on-cluster experience.

2. **10 MCP servers from day one — Red Hat, partners and community**: Three Red Hat MCPs (OpenShift, Ansible Automation Platform, Insights/Lightspeed), five partner MCPs (Confluent Kafka, EnterpriseDB pg-airman-mcp, HashiCorp Terraform, Microsoft Azure, Dynatrace) and two community MCPs (MongoDB, MariaDB) ship pre-loaded, spanning infrastructure automation, data, observability and cloud operations.

3. **This is just the beginning**: The MCP Catalog is the first step in a full MCP ecosystem buildout — quickstarts for featured partners are coming next, alongside supply chain management, trust tiering, governance, access control and lifecycle automation.
