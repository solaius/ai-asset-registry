Design Decisions

  • Follows MLflow version detail exactly: Breadcrumb → Title + lifecycle dropdown → Metadata → Description → Tags → Schema
  • Deployment section — MCP-specific. Shows namespace, endpoint, and gateway registration status. No MLflow equivalent.
  • Server Definition (server.json) — MCP-specific. The immutable payload displayed as syntax-highlighted JSON. Analogous to MLflow's artifact reference.
  • Tools Schema table — maps to MLflow's Input/Output Schema section. Shows tool name, description, and input schema per tool.
  • Source Run link — in metadata, follows MLflow's run lineage pattern
  • Lifecycle dropdown in header — same stage transition pattern as MLflow (Transition to → [state])

3.next Additions

  • Governance Status panel — below Deployment section showing Approval (Pending/Approved/Rejected), Verification (Unverified/Verified), Certification (None/Certified) as badge rows with timestamps and decision rationale
  • "Request Approval" button in header
  • Audit trail — collapsible section showing state change history with who/when/why
