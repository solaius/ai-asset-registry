Design Decisions

  • Dropdown mirrors MLflow's stage transition dropdown — current state badge + chevron, opens a menu showing valid target states with descriptions
  • Only valid transitions shown — Draft can go to Published; Published can go to Deprecated; Deprecated can go to Retired. Invalid paths are not offered.
  • Confirmation modal — matches MLflow's transition confirmation with: visual state change arrow, info alert explaining impact, optional comment, and checkbox to deprecate previous versions
  • "Deprecate all other published versions" checkbox — mirrors MLflow's "archive existing versions in Production" pattern
  • Context-sensitive button label — "Publish" when transitioning to Published, "Deprecate" when deprecating, etc.

3.next Additions

  • Candidate state — between Draft and Published, requiring approval before publishing
  • Approval gate — "Submit for Approval" button instead of direct "Publish" when approval workflow is enabled
  • Required comment — comment becomes mandatory for certain transitions (Deprecate, Retire)
