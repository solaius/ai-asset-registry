Design Decisions

  • Modal dialog — matches MLflow's "Create Model" pattern (not a separate page)
  • Fields: Name (required), Description, Source Type toggle (Internal/External), server.json definition (required), Tags
  • server.json input — three methods: Paste JSON, Upload File, From URL. This is MCP-specific (MLflow has a simple name+description form)
  • Syntax-highlighted JSON preview in a dark code editor block — shows the server definition clearly
  • Internal/External toggle — segmented control distinguishing in-cluster (Lifecycle Operator) vs external MCPs
  • Info banner — tells the user what will happen ("creates Version 1 in Draft state")
  • Immutability note — server.json becomes immutable once registered (key data model invariant)

3.next Additions

  • Workspace selector — choose which workspace/scope to register in
  • Auto-submit for approval checkbox — "Submit for approval after registration"
  • Policy association — link governance policies at registration time
  • Source verification — verify source repo/image before accepting
