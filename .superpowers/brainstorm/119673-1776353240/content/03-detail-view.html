Description

Connect your AI agents to OpenShift clusters for querying cluster state, managing workloads, and troubleshooting deployments through natural language. When a pod fails at 2 a.m., your on-call engineer asks the agent for the last 50 log lines and the resource status of the affected deployment. No context-switching between dashboards, no kubectl gymnastics.

Design Decisions

  • Follows MLflow model detail pattern exactly: Breadcrumb → Title + Actions → Metadata → Tags → Description → Versions (all collapsible sections)
  • Lifecycle state dropdown in header — mirrors MLflow's stage transition dropdown (badge + chevron)
  • Overflow menu (⋮) — for Delete action, matching MLflow pattern
  • Tools section — MCP-specific, no MLflow equivalent. Shows tool name + description in a 2-column grid with "Show all" expand
  • MCP-specific metadata fields: Transport, Endpoint, Workspace (MLflow has Source Run, Copied From)
  • Tags use key:value format — matches MLflow's editable tag pattern
  • Version table — matches MLflow's version table with status dot, version link, registered date, lifecycle state, creator, description
  • Lifecycle states shown: Published (lime/green), Deprecated (lemon/yellow), Retired (charcoal/dark)

3.next Additions (when toggle is ON)

  • Governance panel below metadata — shows Approval Status, Verification Status, Certification Status as badge rows
  • Deployment References section — linked deployments with gateway registration status
  • Approval Status column in version table
  • "Request Approval" button in header next to lifecycle dropdown
