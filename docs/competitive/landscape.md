# Competitive Landscape

## MCP Catalogs & Registries

| Name | Type | Key Differentiator | Scale |
|---|---|---|---|
| **Smithery** | Open marketplace | Largest open marketplace; CLI/SDK | Thousands of capabilities |
| **Docker MCP Catalog** | Curated + containerized | Verified Docker images; Docker Desktop integration | 100+ verified tools |
| **GitHub MCP Registry** | Open registry | GitHub identity & curation; announced Sep 2025 | Official registry |
| **Official MCP Registry** | Protocol-official | "App store for MCP servers"; live API docs | Growing |
| **ToolHive (Stacklok)** | Open + secure runtime | Security-focused; sandboxed containers; CLI/API | Focus on security |
| **Microsoft Copilot Studio** | Closed vendor | In-product certified connectors | Microsoft ecosystem |
| **Databricks Unity Catalog** | Enterprise | Unified governance for data + AI | Enterprise standard |
| **MCP.so** | Third-party directory | Large aggregator | Claims 16K+ servers |
| **Pulse MCP** | Directory | Daily-updated directory | Reports 6K+ servers |

## Red Hat Differentiator

Most competitors are **discovery/listing** focused. Red Hat's value proposition is **enterprise governance**:

1. **Governance-first**: Not just listing servers, but governing their lifecycle, approval, policy, and trust
2. **Platform-integrated**: Deep integration with OpenShift, Gateway API, Kuadrant, RBAC
3. **Runtime enforcement**: MCP Gateway provides actual policy enforcement, not just metadata
4. **Multi-asset**: Registry framework extends beyond MCP to agents, models, prompts, skills
5. **Enterprise features**: RBAC, multi-tenancy, promotion workflows, audit trails, supply chain security
6. **Federated**: Sync with external registries while maintaining internal governance

## Key Distinction: Catalog vs Registry vs Marketplace

| Type | Purpose | Examples |
|---|---|---|
| **Directory/Marketplace** | Discovery only | Smithery, MCP.so, Pulse MCP |
| **Registry** | Governed, runnable assets | Docker MCP, ToolHive, Red Hat MCP Registry |
| **Enterprise Platform** | Full lifecycle governance + runtime | Red Hat (proposed), Databricks Unity |
